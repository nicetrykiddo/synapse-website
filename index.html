Project Overview

Goal ‚Äî Collect & centralize 4 major categories of industrial data for Vasu Breaks (Neemrana) to enable analytics, traceability, and quality monitoring.

Manufacturing Records

Includes batch IDs, machine logs, operator details, and production timestamps.

Testing / QA Data

  • Data validation and cleaning

This ensures seamless analysis across manufacturing, quality, customer service, and sales operations.

Our data centralization system leverages core OOP principles to process 10,700 records across 4 datasets with clean, maintainable, and scalable architecture.

Concept

Classes serve as blueprints defining structure and behavior, while objects are instances created from these blueprints.

    private String productId;
    private String serialNo;
    private Date failureDate;
    private String location;
    private String failureMode;
    private String severity;
    private String reportedBy;

    // Constructor, getters, setters...
}

Concept

Inheritance allows child classes to acquire properties and methods from parent classes, promoting code reusability and establishing hierarchical relationships.

Base Class & Inheritance Example 
// Base class with common properties
public abstract class DataRecord {
    protected String recordId;
    protected String productId;
    protected Date timestamp;

    public abstract void validate();
}

// Child class inherits from base
public class FieldReport extends DataRecord {
    private String failureMode;
    private String severity;
    private String location;
    // Inherits: recordId, productId, timestamp
}

Inheritance Hierarchy

Benefit

Eliminated code duplication by inheriting 3 common fields across 10,700 records. Centralized shared behavior while maintaining dataset-specific functionality.

Concept

Polymorphism enables objects of different classes to be treated uniformly through a common interface, with each class providing its own implementation of shared methods.

Polymorphic Method Example 
// Base class defines the contract
public abstract class DataRecord {
    public abstract boolean validate();
}

// Each class implements validation differently
public class FieldReport extends DataRecord {
    @Override
    public boolean validate() {
        return isValidFailureMode(failureMode)
            && isValidLocation(location);
    }
}

public class TestingData extends DataRecord {
    @Override
    public boolean validate() {
        return measurementValue >= thresholdMin
            && measurementValue <= thresholdMax;
    }
}

Method Implementations

Benefit

Processed all 10,700 records through unified pipeline. Single processing loop handles different data types without type-checking logic, enabling clean and maintainable code.

Concept

Abstraction hides complex implementation details and exposes only essential features, defining what operations must be performed without specifying how.

Abstract Class Example 
public abstract class DataProcessor {
    // Abstract methods - must be implemented
    public abstract void readData();
    public abstract void cleanData();
    public abstract void validateData();
    public abstract void exportData();

    // Template method - defines workflow
    public void processData() {
        readData();
        cleanData();
        validateData();
        exportData();
    }
}

Abstract Methods & Implementations

Abstract Method Purpose Implementations
readData() Load data from source CSVReader, DatabaseReader
cleanData() Remove nulls & standardize NullRemover, FormatStandardizer, DuplicateDetector
validateData() Ensure quality criteria CompletenessValidator, ValidityChecker, ConsistencyChecker
exportData() Write to destination CSVExporter, JSONExporter, DatabaseExporter

Benefit

Established consistent ETL pipeline contract across all datasets. Achieved 100% data completeness and quality score by enforcing validation at abstract level, while allowing flexible implementation per dataset type.

Probability Concepts Used in the Project

Explore the key probability distributions and theorems applied in our data analysis workflow.

Normal Distribution

Gaussian Distribution

Definition

A normal distribution is also known as the Gaussian distribution. It describes a symmetrical curve where most values cluster around the mean with symmetric variation.

Purpose

Used to analyze natural variation in continuous data such as sales values, product dimensions, or manufacturing time.

Poisson Distribution

Event Count Model

Definition

A Poisson distribution models the probability of a number of events occurring within a fixed time or space interval. The Poisson distribution is defined by a single parameter, lambda, where lambda is greater than zero.

Purpose

Used to study the count of events such as number of daily orders, number of defects per batch, or machine breakdowns.

Exponential Distribution

Time Between Events

Definition

An exponential distribution describes the time gap between consecutive events. It is defined by a single parameter, lambda, which is called the rate parameter.

Purpose

Helps analyze time between orders, customer arrivals, or time until machine failure.

Conditional Probability

Dependent Events

Definition

Conditional probability measures the chance of an event occurring given that another event has already taken place.

Purpose

It helps in exploring the dependencies between variables (e.g. if marketing spend increases, what is the probability that sales will also increase?, Does Machine A produce more defects compared to Machine B?) to understand cause-effect relationships and uncover patterns that influence outcomes.

Bayes' Theorem

Inverse Probability

Definition

Bayes' theorem is a mathematical rule that helps us recalculate the probability of something by combining old information with new evidence, making predictions more accurate.

Purpose

Bayes' theorem helps update the probability of events when new data appears. It improves predictions like customer conversion chances or machine failure risk by combining old information with new evidence.
